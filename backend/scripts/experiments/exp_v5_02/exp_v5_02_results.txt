================================================================================
Date: 2026-03-07

HYPOTHESIS
----------
Adding delay alongside the IIR filter (from Exp 1) should resolve the
confounding that pure delay (v4) couldn't. The delay+filter combination
provides both temporal shift and smoothing, potentially breaking the
d <-> pao2_0 degeneracy.

DATA
----
6 holds from session 1, all per-hold fits (no global parameters).

  FL#1:   Full Lung hold #1
  FL#6:   Full Lung hold #6
  FRC#2:  Functional Residual Capacity hold #2
  FRC#5:  Functional Residual Capacity hold #5
  RV#3:   Residual Volume hold #3
  RV#4:   Residual Volume hold #4

Lung volume types:
  FL  — Full Lung: maximal inspiration, highest initial PAO2, slowest desat
  FRC — Functional Residual Capacity: normal resting volume, moderate initial PAO2
  RV  — Residual Volume: maximal expiration, lowest initial PAO2, fastest desat


================================================================================
MODEL
================================================================================

Physiology (common to all mechanistic variants):
  PAO2(t)  = pvo2 + (pao2_0 - pvo2) * exp(-t / tau_washout)
  PaCO2(t) = paco2_0 + k_co2 * t
  P50(t)   = 26.6 + 0.48 * (PaCO2(t) - 40)         [Bohr effect]

ODC (Severinghaus + gamma):
  PAO2_virtual = PAO2 * (26.6 / P50)
  PAO2_adj     = 26.6 * (PAO2_virtual / 26.6)^gamma
  SaO2         = 100 / (1 + 23400 / (PAO2_adj^3 + 150 * PAO2_adj))

Sensor (Delay+Filter variant):
  SaO2_delayed = interp(t - d, t, SaO2, left=SaO2[0])
  alpha        = dt / (tau_f + dt),  dt = 1s
  SpO2[i]      = (1 - alpha) * SpO2[i-1] + alpha * SaO2_delayed[i]
  SpO2         = clip(SpO2 + r_offset, 0, 100)

Richards (phenomenological baseline):
  SpO2 = s_min + (s_max - s_min) / (1 + nu * exp((t - t50)/k))^(1/nu)

Loss function:
  Weighted SSE: w(i) = 3.0 if SpO2_obs(i) < 95%, else 1.0
  Richards uses unweighted SSE.

================================================================================
KEY FINDINGS
================================================================================

1. d PINS AT UPPER BOUND (30s) for most holds in both Delay and D+F variants.
   This confirms the d <-> pao2_0 confound from v4 persists even with the
   addition of an IIR filter. On a monotonically declining SpO2 curve,
   shifting the signal in time (delay) is nearly equivalent to adjusting
   the initial alveolar oxygen (pao2_0) or washout time constant (tau_washout).

2. ADDING FILTER TO DELAY (D+F, 9p) provides NEGLIGIBLE improvement over
   delay-only or filter-only (both 8p) on apnea data. The extra parameter
   does not buy meaningful information. Both delay and filter produce similar
   effects on a monotone declining signal: they shift and/or smooth it, which
   are nearly interchangeable operations on this class of curves.

3. GAMMA BEHAVIOR reveals the narrowed-bounds workaround:
   - Delay variant: gamma = 1.3-1.5 (forced by [0.8, 1.5] bounds)
   - All other variants: gamma = 1.5-2.0 (free to inflate with [0.8, 2.0])
   The reduction in gamma for the Delay variant is an artifact of tighter
   bounds, not evidence that the delay model is absorbing the ODC distortion.

4. TAU_F IN D+F IS HIGHLY VARIABLE across holds — it is not a stable,
   identifiable parameter on apnea-only data. The optimizer can trade tau_f
   against d, pao2_0, and tau_washout freely. There is no unique optimum.

5. THE DELAY AND FILTER ARE PARTIALLY REDUNDANT on monotone desaturation
   curves. Both operations shift/smooth a declining signal. The data does
   not contain enough structure (inflection points, direction changes) to
   distinguish "the signal is delayed" from "the signal is smoothed."

6. ON APNEA-ONLY DATA, sensor model parameters (d, tau_f) CANNOT be
   separated from physiology parameters (pao2_0, tau_washout). This is a
   structural identifiability problem, not an optimization failure. The
   confounding manifold is inherent in the model+data combination.


================================================================================
CONCLUSIONS
================================================================================

1. CONFIRMED: d <-> pao2_0 confounding persists with delay+filter on
   apnea-only data. The hypothesis that adding filter alongside delay would
   break the confound is REJECTED.

2. The monotone desaturation curve provides INSUFFICIENT INFORMATION to
   identify both d and tau_f. The signal contains only one "feature" — a
   declining curve — which constrains at most one degree of freedom in the
   sensor model. Two sensor parameters (d + tau_f) are over-parameterized
   for this data.

3. A NON-MONOTONE FEATURE (recovery V-shape after breathing resumes) is
   needed to break the symmetry. The SpO2 nadir — where SpO2 continues
   falling briefly after breathing resumes, then recovers — creates an
   asymmetric feature that a pure shift of initial conditions cannot
   replicate.

4. This motivates v5 Experiment 3 / v6 Experiment 1: including RECOVERY
   DATA after breathing resumes, which provides the direction-change
   feature needed to distinguish delay from smoothing and from physiology
   parameter adjustments.

5. Filter-only (Exp 1) and Delay+Filter (Exp 2) perform equivalently on
   apnea data, confirming that the IIR filter is the more parsimonious
   sensor model for monotone descent. Pure delay adds nothing useful
   without recovery data.
